Sass - 시작하기

2021-01-15

✨ Sass 란?

Sass는 "CSS Preprocessor"로 한국어로 하면 CSS 전처기다.
굳이 왜 그냥 CSS 안쓰고 이런 전처리기를 쓰는거냐면
단순한 이유다. 쓰기 편하니까?
복잡한 작업이 쉬워지며, 코드가 재활용 가능하고 가독성을 높여주기 때문이다.

Less, Sass, Stylus 같이 다양한 css 전처리기가 있지만
2020 통계 기준 Sass가 제일 많이 쓰인다.

https://2020.stateofcss.com/en-US/technologies/pre-post-processors/




✨ Sass 사용방법?

그렇다고 무턱대고 그냥 쓸 수는 없고 컴파일을 해야지만 사용 가능하다.
많은 컴파일 방법이 있지만 나는 전 포스팅인 gulp를 사용하여 컴파일한다.




✨ Sass vs. Scss?

원래는 Sass가 먼저 나왔다. 하지만 사용하면서 Sass의 세번째 버전에서 Sass의 모든 기능을 가지며 CSS구문과 완전히 호환할 수 있도록 새로 나온게 Scss이다.

∴ Scss는 CSS와 같은 구문으로 Sass기능을 모두 지원가능하단 말임

이제는 css에서도 사용이 가능하지만 내 기준 Scss의 가장 큰 장점은 변수였다.

2. Gulp 로컬에 설치

npm init는 node 프로그램을 시작하는 명령어로 package.json을 설치한다.
명령어는 내가 실행할 디렉토리에서 실행해야됨!!

그러면 아래 상황들이 나오는데 입맛에 맞게 설정하면 된다.

나는 일단 설치가 목적이기 때문에 엔터만 누른다. (나중에 수정 가능)

$ npm init package name: (test) version: (1.0.0) description: entry point: (index.js) test command: git repository: keywords: author: license: (ISC)

전역에도 설치했지만 모듈을 설치하기 위해서 같은 디렉토리에
아래 명령어를 입력한다.

$ npm install --save-dev gulp



3-1. gulpfile.js 셋팅하기

Gulp를 실행하기 위해선 gulpfile.js를 생성해줘야한다.
자동으로 생기는게 아니기때문에 직접 새 파일로 생성한다.

// gulpfile.js function defaultTask(cb) { console.log('걸프 시작하기') cb(); } exports.default = defaultTask

이제 터미널에 gulp를 입력해주면 아래 처럼 나온다.

$ gulp [14:59:59] Using gulpfile D:\Study\test2\gulpfile.js [14:59:59] Starting 'default'... 걸프 시작하기 [14:59:59] Finished 'default' after 3.2 ms

default는 gulp를 실행하면 시작하는 동작을 말한다.
함수 defaultTask를 실행시켰기 때문에 console.log인 "걸프 시작하기"가 나온다.



3-2. Gulp 메서드

아주아주 기초적인건 설치가 되었다.
근데 우리가 gulp를 사용하기 위해선 메서드를 알아둬야된다.
많은 메서드가 있지만 대표적으로 쓰이는 것만 설명하겠음

src 메서드

작업할 대상 되는 파일을 지정해주는 명령어

gulp.src (globs)
  • globs : 어떤 파일을 읽을 건지 결정, 파일을 선택하는 방식은 node-glob 문법을 따른다.
gulp.src("./src/img/*")



pipe 메서드

gulp의 스트리밍 기능으로 src로 대상을 정했으면 각 파일을 stream 형태로 읽어 다음 플러그인으로 연결해주는 명령어

gulp.src("./src/img/*").pipe(gulp.dest("./dist/img"));



dest 메서드

작업이 종료된 파일의 저장되는 경로를 지정해주는 명령어

gulp.dest(directory, [options])
  • dest : 파일이 기록될 디렉토리
gulp.src("./src/img/*").pipe(gulp.dest("./dist/img"));



watch 메서드

파일의 변경, 업데이트 등을 감지하여 작업을 수행하는 명령어

gulp.watch(globs , [ options ] , [ task ] )
  • globs : 감시할 파일
function css() { return gulp.src('./src/scss/*.scss') .pipe(sass({outputStyle: 'compressed'}).on('error', sass.logError)) .pipe(gulp.dest('./dist/css')); } gulp.watch("./src/scss/**/*", css);



task 메서드

작업을 선언하는 명령어로 이제 더 이상 쓰이지 않는다.



series 메서드

작업들을 조합하며 실행하는 메서드로 순차적으로 실행한다. 직렬구조

gulp.series(...tasks)
  • tasks : 실행할 작업들
gulp.series(css, js,img,html);



parallel 메서드

작업들을 조합하며 실행하는 메서드로 서로 관련없이 실행한다. 병렬구조

gulp.series(...tasks)
  • tasks : 실행할 작업들
gulp.series(gulp.parallel(css, js, img,html));



3-3. sass 설치

이제 원하는 플러그인들을 설치해보자. 우리가 gulp를 사용하는 이유는 다양하지만 제일 큰 이유는 sass를 사용하기 위함이 아닌가 싶다.
sass를 사용하기 위해선 빌드 시스템이 필요하기 때문이다.

나는 dartSass를 설치한다. 이유는 없다 그냥 따라해라 (나중에 설명함)

아래와 같이 선언해주면 된다.

  • gulp-dart-sass의 자세한 사용방법
  • sass 옵션
$ npm install gulp-dart-sass --save-dev // gulpfile.js const gulp = require('gulp'); //require()은 ES6의 import랑 같은 메서드 const sass = require('gulp-dart-sass'); function css() { return gulp.src('./src/scss/*.scss') .pipe(sass({outputStyle: 'compressed'}).on('error', sass.logError)) .pipe(gulp.dest('./dist/css')); }



3-4. babel 설치

그 다음은 javascript ES6버전이후를 사용하기 위한 babel을 설치해준다.

아래와 같이 선언해주면 된다.

  • gulp-babel의 자세한 사용방법
  • babel 옵션
$ npm install --save-dev gulp-babel @babel/core @babel/preset-env // gulpfile.js const gulp = require('gulp'); //require()은 ES6의 import랑 같은 메서드 const babel = require('gulp-babel'); function js(){ return gulp.src('./src/js/*.js') .pipe(babel()) .pipe(gulp.dest('./dist/js')) }
